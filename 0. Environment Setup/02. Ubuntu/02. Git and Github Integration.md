# Git and GitHub Integration (Ubuntu)

Step-by-step guide to install Git, connect to GitHub, and use VS Code for everyday Git work on Ubuntu.

## 1) Install Git
- Update package lists and install:
  ```bash
  sudo apt update
  sudo apt install git
  ```
- Verify installation:
  ```bash
  git --version
  ```

## 2) Global Git configuration
Run in Terminal (replace with your details):
```bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global core.editor "code --wait"   # optional: use VS Code for Git messages
```

## 3) Sign in or create a GitHub account
- Go to https://github.com and sign in (or create an account).
- Add a recognizable profile email that matches `user.email` if you want verified commits.

## 4) Authenticate Git with GitHub
Pick one method (SSH is recommended for long-term use).

### Option A: SSH keys (recommended)
1. Check for existing keys:
   ```bash
   ls -al ~/.ssh
   ```
2. Generate a new key (preferred: ed25519):
   ```bash
   ssh-keygen -t ed25519 -C "you@example.com"
   # If ed25519 is unavailable:
   # ssh-keygen -t rsa -b 4096 -C "you@example.com"
   ```
3. Start the ssh-agent and add your key:
   ```bash
   eval "$(ssh-agent -s)"
   ssh-add ~/.ssh/id_ed25519
   ```
4. Copy the public key to clipboard (install xclip if needed):
   ```bash
   sudo apt install xclip           # only if xclip is not installed
   xclip -sel clip < ~/.ssh/id_ed25519.pub
   ```
5. Add it to GitHub: **Settings → SSH and GPG keys → New SSH key** → paste → Save.
6. Test the connection:
   ```bash
   ssh -T git@github.com
   ```

### Option B: Personal Access Token (HTTPS)
1. In GitHub: **Settings → Developer settings → Fine-grained tokens** (or **Personal access tokens → Tokens (classic)**).
2. Create a token with `repo` scope (and `workflow` if needed). Copy it once generated.
3. When running `git clone`/`git push` over HTTPS, use your GitHub username and paste the token as the password.
4. Enable a credential helper to cache tokens (pick one):
   ```bash
   git config --global credential.helper store      # saves token in plaintext (~/.git-credentials)
   # or if you install Git Credential Manager:
   # sudo dpkg -i git-credential-manager_*_amd64.deb
   # git-credential-manager configure
   # git config --global credential.helper manager
   ```

## 5) VS Code integration
- Install VS Code (deb package or apt repo): https://code.visualstudio.com/
- Sign in to GitHub from VS Code (Accounts icon in the bottom-left or `F1` → "GitHub: Sign in").
- Set your preferred shell as the default terminal (`F1` → "Terminal: Select Default Profile").
- The Source Control panel (`Ctrl+Shift+G`) handles staging, committing, branching, and viewing diffs.

## 6) Common workflows
- **Clone a repository**
  ```bash
  git clone git@github.com:USERNAME/REPO.git
  # or HTTPS
  # git clone https://github.com/USERNAME/REPO.git
  ```
- **Initialize a new repository**
  ```bash
  git init
  echo ".venv/" >> .gitignore   # example ignore entry
  git add .
  git commit -m "Initial commit"
  git branch -M main
  git remote add origin git@github.com:USERNAME/REPO.git
  git push -u origin main
  ```
- **Keep your branch updated**
  ```bash
  git pull origin main
  ```
- **Create and push a feature branch**
  ```bash
  git switch -c feature/brief-description
  git add .
  git commit -m "Describe the change"
  git push -u origin feature/brief-description
  ```

## 7) Troubleshooting tips
- `git: command not found`: run `sudo apt install git`, then reopen the terminal.
- Permission denied (SSH): ensure the key is added to ssh-agent and to GitHub, then rerun `ssh -T git@github.com`.
- Clipboard step fails for SSH key: install `xclip` and retry, or print the key with `cat ~/.ssh/id_ed25519.pub`.
- Re-auth prompts over HTTPS: set a credential helper; regenerate a token if expired.
- Wrong author info in commits: update `user.name` and `user.email`, then amend the commit if needed.
