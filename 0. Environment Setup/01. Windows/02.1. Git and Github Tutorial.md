## 11. Everyday Git Workflow in VS Code

### 11.1. Make Changes

* Edit files in VS Code.
* New or modified files appear in **Source Control → Changes**.

### 11.2. Stage Changes

* In the **Source Control** panel:

  * Click the **+** icon next to each file to stage them, or
  * Click **Stage All Changes**.

### 11.3. Commit Changes

1. After staging, write a commit message in the input box at the top, for example:

   ```text
   Add login page UI
   ```

2. Click **✔ Commit** or press `Ctrl + Enter`.

### 11.4. Push to GitHub

* In the **Status Bar** at the bottom, click the icon showing up/down arrows.
* Or open the Command Palette and run:

  ```text
  Git: Push
  ```

VS Code will push your commits to GitHub.

### 11.5. Pull Latest Changes from GitHub

If someone else (or you from another machine) pushed changes:

* In the Command Palette, run:

  ```text
  Git: Pull
  ```

* Or click the **arrow down** icon in the Status Bar.

## 12. Basic Git Commands (Optional, in Terminal)

You can also run Git commands inside VS Code:

1. Open the built-in terminal:
   **View → Terminal** or press `` Ctrl + ` ``

2. Verify Git:

   ```bash
   git status
   ```

3. Common commands:

   ```bash
   # See status
   git status

   # Stage all files
   git add .

   # Commit with message
   git commit -m "Your message here"

   # Push to remote
   git push

   # Pull from remote
   git pull
   ```

## 13. Creating and Switching Branches in VS Code

### 13.1. Create a New Branch

1. Click the **branch name** in the Status Bar (usually `main` or `master`).
2. Select **Create new branch…**.
3. Enter a branch name, e.g. `feature-login`.
4. VS Code will switch you to this new branch.

### 13.2. Switch Branches

* Click the branch name in the Status Bar.
* Choose another branch from the list.
