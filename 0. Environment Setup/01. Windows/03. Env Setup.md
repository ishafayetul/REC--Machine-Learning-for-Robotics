This project uses a local virtual environment for all Python and ML work on **Windows**.



### Create & Activate

From the project root, open **PowerShell** or **Command Prompt**.

#### Option 1 â€“ PowerShell (recommended)


#### Option 2 â€“ Command Prompt (cmd)

```bat

```

After activation, your prompt should show something like:

```text
(.venv) C:\path\to\project>
```

---

### Install Core Dependencies

With the virtual environment **activated**:

```bash
pip install --upgrade pip
pip install numpy pandas matplotlib scikit-learn jupyter ipykernel
```

---

### Install PyTorch with GPU Support (Only for Deep Learning)

> Make sure your NVIDIA driver and CUDA toolkit are properly installed if you want GPU support.

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

---

### Register Jupyter Kernel

Still inside the activated `.venv`:

```bash
python -m ipykernel install --user --name=ml --display-name="ML (.venv)"
```

---

### Install VS Code Extensions

In **VS Code**, install these extensions:

* **Python** (by Microsoft)
* **Jupyter** (by Microsoft)

---

### Select the Interpreter in VS Code

1. Press:
   `Ctrl + Shift + P`
2. Type:
   `Python: Select Interpreter`
3. Choose the interpreter that points to:

```text
.your-project-folder\.venv\Scripts\python.exe
```

(or similar, with `.venv` in the path)

---

### Quick Verification

Open **Python** (REPL) or a **Jupyter notebook** inside the activated `.venv` and run:

```python
import sys, numpy, pandas, sklearn
print(sys.executable)
print(numpy.__version__)
print(pandas.__version__)
print(sklearn.__version__)
```

You should see:

* `sys.executable` pointing to `.venv\Scripts\python.exe`
* Version numbers printed without errors

If that works, your Windows ML dev environment is ready. âœ…
