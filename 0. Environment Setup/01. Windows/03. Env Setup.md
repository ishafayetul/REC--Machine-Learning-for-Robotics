# Python & ML Environment Setup (Windows)

Set up a clean Python environment for machine learning work on Windows using VS Code and `venv`.

## 1) Install Python 3.10+ (with PATH)
- Download from https://www.python.org/downloads/windows/ (64‑bit installer).
- During setup: check **Add python.exe to PATH** and choose **Customize installation** to keep `pip` enabled.
- Verify in a new terminal (PowerShell or Command Prompt):
  ```bash
  python --version
  pip --version
  ```

## 2) Create a workspace folder
Pick a short path without spaces (e.g., `C:\ml-workspace`) to avoid toolchain issues.

## 3) Create a virtual environment
From the workspace folder:
```bash
python -m venv .venv
.\.venv\Scripts\activate   # PowerShell or Command Prompt
```
- The prompt should show `(.venv)` when active.
- Deactivate anytime with `deactivate`.

## 4) Upgrade pip and install core packages
Run inside the activated environment:
```bash
python -m pip install --upgrade pip
pip install numpy pandas matplotlib scikit-learn jupyterlab ipykernel
```
- Add project-specific packages later with `pip install <package>`.

## 5) Configure VS Code
- Install VS Code: https://code.visualstudio.com/
- Extensions to install: **Python**, **Jupyter**, **Pylance**.
- Open the workspace folder in VS Code, then select the interpreter: `Ctrl+Shift+P` → “Python: Select Interpreter” → choose the `.venv` interpreter.
- For notebooks, VS Code will auto-detect `.venv`; if not, pick it via the kernel picker.

## 6) Use Jupyter notebooks (optional)
- Start JupyterLab from the workspace:
  ```bash
  jupyter lab
  ```
- Or create `.ipynb` files directly in VS Code and run cells there.

## 7) GPU acceleration (optional, NVIDIA)
- Ensure the latest NVIDIA driver is installed (GeForce Experience or https://www.nvidia.com/Download/index.aspx).
- Install CUDA toolkit only if a library explicitly requires it; many packages ship their own runtimes.
- To install GPU PyTorch (example, adjust per your CUDA/driver):
  ```bash
  pip install torch --index-url https://download.pytorch.org/whl/cu121
  ```
- Verify availability inside Python:
  ```python
  import torch
  print(torch.cuda.is_available(), torch.cuda.get_device_name(0))
  ```

## 8) Common troubleshooting
- **`python` not found**: reopen the terminal or reinstall Python with PATH enabled.
- **Wrong interpreter in VS Code**: re-run “Python: Select Interpreter” and pick `.venv`.
- **Package install fails with permissions**: ensure the virtual environment is active; avoid `pip` with admin PowerShell.
- **Pip uses old cache**: `pip install --upgrade --no-cache-dir <package>`.
- **Corrupt venv**: delete `.venv/` and recreate with `python -m venv .venv`, then reinstall dependencies.
